Per-realisation random LOS sky positions
=========================================

Goal: For multi-realisation fields (Manticore, CB1, CB2, CF4, HAMLET),
each realisation should use independent random sky positions instead of
sharing the same 1000 directions.

Current state
-------------
- compute_los.py generates RA/dec once with seed=42.
- HDF5 output: RA(n_gal,), dec(n_gal,), los_density(n_sims, n_gal, n_r).
- Models assume RA/dec are 1D (shared across realisations).
- base_model.py asserts data[rand_los_RA].ndim == 1 (line 167).
- rhat_rand_los is (n_los, 3), used in Vext projection as
  Vext[None,:] * rhat_rand_los (lines 485, 234 in base/TRGB models).
- Subsampling (max_rand_los) indexes RA/dec with 1D idx (line 211).


Files to change
---------------

1. scripts/preprocess/compute_los.py
   - Extract random sky generation into a helper:
       generate_random_sky(npoints, seed) -> RA, dec
   - In the per-realisation loop, if catalogue is random and n_sims > 1,
     generate RA_i, dec_i with seed = 42 + nsim.
   - Gather RA_i, dec_i per realisation.
   - Save RA as (n_sims, n_gal) and dec as (n_sims, n_gal) in HDF5.
   - For single-realisation fields, keep RA/dec as (n_gal,) unchanged.

2. candel/pvdata/data.py  —  load_los()

   - But the mask branch (line 473) does f["RA"][...][mask], which

     is safe. Add a comment noting this.

Notes
-----
- The seed convention seed=42+nsim ensures reproducibility while giving
  each realisation independent positions.
- For single-realisation fields (Carrick, Lilow, CLONES) the behavior
  is unchanged: RA/dec remain 1D with seed=42.
- Memory impact is small: 30 * 1000 * 2 floats = 240 KB extra.
